#  Customer Churn Analysis & Retention Strategy

## 📝 Project Overview
This project focuses on analyzing customer behavior to identify patterns and reasons behind customer **Churn**. As a Financial Analyst, I've integrated financial metrics with operational data to predict which customers are likely to leave and provide recommendations to improve retention and **Lifetime Value (LTV)**.

## 🎯 Objectives
- Identify key factors contributing to customer churn.
- Visualize churn trends using **Power BI**.
- Provide data-driven recommendations for the marketing and sales teams.
- Calculate the **Financial Impact** of lost customers.

## 🛠 Tech Stack
- **Python:** Data Cleaning and Exploratory Data Analysis (EDA).
- **SQL:** Querying customer transaction data.
- **Power BI:** Building an interactive dashboard for stakeholders.
- **Excel:** Initial data auditing and pivot tables.

## 📑 Key Insights
- **Feature Importance:** Discovered that [X factor, e.g., contract length] is the primary driver of churn.
- **Financial Impact:** Churn resulted in a [X%] loss of recurring revenue last quarter.
- **Segment Analysis:** High-value customers have a lower churn rate but a higher impact when they leave.

## 📈 Dashboard Features
- **Churn Rate Overview:** Monthly and Quarterly trends.
- **Customer Segmentation:** Breakdown by demographics and service usage.
- **Predictive Metrics:** Identification of "At-Risk" customer segments.

## 📁 How to use
1. Clone this repository.
2. Open the `analysis.ipynb` to see the Python cleaning process.
3. Open the `Churn_Dashboard.pbix` to interact with the visual report.
